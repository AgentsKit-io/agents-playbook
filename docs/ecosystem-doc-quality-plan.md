# Playbook ecosystem documentation quality plan

## Outcome

Make the Playbook the reference implementation of the shared ecosystem continuation pattern: one concise narrative, one seven-product manifest, six contextual exits, and complete human and agent discovery surfaces.

## Acceptance contract

- The canonical manifest contains the seven ecosystem products in workflow order.
- The Playbook ecosystem section renders exactly six peers and remains usable at 375, 768, 1280, and 1440 px.
- Documentation, chat, review, and enterprise language links respectively to Doc Bridge, AgentsKit Chat, Code Review, and AKOS at the point of need.
- `/for-agents`, `/llms.txt`, `/llms-full.txt`, raw Markdown, sitemap, README, and deterministic discovery agree on canonical URLs.
- The docs stay concise: visual cards and a small routing table replace duplicated prose.
- Doc Bridge remains exactly 100/100 and the existing product, unit, build, and E2E gates pass.

## Delivery slices

1. Replace the stale four-product registry with the canonical seven-product contract.
2. Expand the visual continuation section and compact hero cross-reference to six peers.
3. Add a dedicated `for-agents` guide and route, then align README, docs, LLM artifacts, sitemap, and Ask Playbook discovery.
4. Add contract tests for identity, order, canonical routes, six-peer discovery, and machine surfaces.
5. Certify responsiveness, accessibility, build output, deterministic artifacts, and Doc Bridge health.

## Non-goals

- Reimplementing AgentsKit Chat behavior in this host.
- Adding long-form marketing copy.
- Publishing, committing, pushing, or opening a pull request in this stage.
